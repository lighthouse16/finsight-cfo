## 2. Source Modes & Provenance Definition

All endpoints and data structures in the Market Funding Intelligence module explicitly identify data origin according to these 6 normalized modes:

1. **`live`**: Public HTTP APIs (HKMA / HKAB / FX) parsed live with safe fallbacks.
2. **`provider_configured`**: Real commercial data adapters fully configured with valid keys.
3. **`provider_not_configured`**: Active commercial adapter slots lacking license keys.
4. **`workspace_derived`**: System-calculated scores/benchmarks based on company parameters.
5. **`fixture`**: Local JSON mock catalog or seed benchmarks.
6. **`unavailable`**: Component is offline or not configured.

Each market data value carries: source name, source mode, `as_of` timestamp, freshness, caveat, and confidence score.

## 3. SME Financial Data Room Ingestion Details

The workspace-backed persistent Data Room ingestion implementation covers the following detailed capabilities:
- **Permanent File Storage**: Handled by `FileStore.save_file` on disk (local mode) or DB blob storage.
- **Real OCR Ingestion**: Apache Tika-based extraction for raw PDF/DOCX files, with structured parser fallback for CSV/XLSX.
- **Database Metadata Persistence**: Stored via JSON attributes on file record schemas.
- **Dynamic Parser Run**: Structured statements parsed on upload using CSV/XLSX modules.
- **Analysis Rerun**: Executes ratios, risk diagnostics, projections, and DCF valuation on the compiled active snapshot on-the-fly.
